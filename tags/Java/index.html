    单例模式

    2017-04-01
    Java

    概念

    单例：即只有一个实例对象

    单例的模式：

    • 饿汉式：效率低、线程安全、在第一次加载类时自动创建实例对象
    • 懒汉式：第一次调用实例对象时，将实例对象创建并返回，线程不安全，效率高

    如何实现单例：

    1. 构造器私有化
    2. 提供一个私有的静态的成员变量，并且自行创建该实例对象
    3. 提供一个静态的公共的访问方法、将该实例对象返回调用者

    为了保护线程安全，使用synchronized

    饿汉式

    提供一个静态的、私有的、最终的该实例对象，并且赋值，这种方法在某些特定的情况下，占用内存

    策略设计模式

    2017-03-30
    Java

    策略设计模式

    软件的设计模式，指的是由某个行为，但是在某个场景中，该行为有不同的实现算法。

    实现方法

    • 定义一个接口，创建功能方法。
    • 各个策略类实现接口，并在功能方法中实现各自不同的逻辑。
    • 在调用者类中创建一个接口对象，重写调用者构造器，将传进来的策略类的对象赋值给接口对象，并在之后的功能方法中调用策略类对象的功能方法

    比如：每个人都得交税：

    —–>美国税 %4

    —–>中国 %20

    不一样—–> 税收算法不一样

    模板方法

    2017-03-30
    Java

    模板方法设计模式：

    在父类的一个方法中定义一个总体的算法骨架，而将某一些步骤延迟到子类中去，不同的子类实现的细节不同，模板方法是的子类可以不改变父类的算法结果，而使得算法的细节步骤改变，抽象父类负责定义操作中的总体的算法骨架，某些具体实现步骤由子类决定

    注意：
    抽象的父类，只是定义了一个通用的算法，其实现必须依赖子类的辅助，模板方法作为模板样式不准子类复写

    接口

    2017-03-26
    Java

    概念

    表示一种规范，用来定义一组方法，表示某一类事务具备的功能，其实现类必须实现该功能。

    注意

    • 使用interface关键字声明
    • 接口能实例化
    • 一个类可实现（implement）多个接口（中间用逗号隔开），但只能继承一个类
    • 接口中的方法默认使用public abstract修饰
    • 接口中的成员变量public static final修饰
    • 接口中不能实现接口，但允许继承多个接口
    • 接口多继承时，实现最终接口的类必须把所有的父接口的方法实现

    多态

    2017-03-25
    Java

    概念：

    多种形态，是指父类或接口定义的引用变量，可以只想子类或者实现子类的实例对象，在调用方法时，是调用该引用变量所指向的实例对象。

    满足多态的三个条件：

    1. 继承
    2. 重写
    3. 父类引用指向子类实例对象

    注意：父类引用对象不能使用子类新增方法

    父类与子类关系

    • 父类是父类
    • 子类是父类
    • 子类是子类
    • 子类不是其他类
    • 父类不是子类

    类对象

    2017-03-22
    Java

    类成员（static 修饰成员）的特点：

    1. 随着类被加载进JVM 而同时也初始化和内存中分配空间
    2. 优先于对象存在（对象也是通过new 出来的）
    3. 被该类的所有对象所共享
    4. 直接使用类名调用即可

    访问规则：

    1. 静态方法：只能访问静态的成员变量/方法 不能访问非静态的成员变量/方法
    2. 非静态方法可以访问非静态成员变量/方法 也可以访问静态的成员变量/方法
    3. 对象可以访问静态成员、但是不推荐、因为底层依然是同类名在调用
    4. 类可以访问静态成员、类不能访问非静态成员（成员变量/方法）
