    Java

    封装的概念：

    1. 把对象的状态和行为看成一个统一的整体，将而这存放在一个独立的模块中（类中）
    2. 信息隐藏：把不需要让外界知道的信息隐藏起来，尽可能的而隐藏对象的功能实现细节

    问题：

    1. 封装什么类？
    2. 封装什么行为和状态？
    3. 如何隐藏信息？

    接口

    2017-03-26
    Java

    概念

    表示一种规范，用来定义一组方法，表示某一类事务具备的功能，其实现类必须实现该功能。

    注意

    • 使用interface关键字声明
    • 接口能实例化
    • 一个类可实现（implement）多个接口（中间用逗号隔开），但只能继承一个类
    • 接口中的方法默认使用public abstract修饰
    • 接口中的成员变量public static final修饰
    • 接口中不能实现接口，但允许继承多个接口
    • 接口多继承时，实现最终接口的类必须把所有的父接口的方法实现
    public class InterfaceDemo03 {
    public static void main(String[] args) {
        //提供键盘对象、鼠标对象
        MainBoard.plugin(new Mouse());
        MainBoard.plugin(new KeyBoard());
        MainBoard.doWork();
    }
}
interface IUSB{
    //提供一个交换数据
    void swapData();
}
//鼠标
class Mouse implements IUSB{
    @Override
    public void swapData() {
        System.out.println("鼠标在移动。。。");
    }
}
//硬盘
class KeyBoard implements IUSB{

    @Override
    public void swapData() {
        System.out.println("敲打键盘。。。");
    }
}
//主板
class MainBoard{
    //声明成员变量
    private static IUSB[] usbs = new IUSB[2];
    //声明一个控制当前下标的变量
    private static int index = 0;
    //写一个静态方法，用于将鼠标、键盘插入
    public static void plugin(IUSB iusb){
        if (usbs.length == index){
            System.out.println("USB满了");
            return;
        }
        usbs[index] = iusb;
        index++;
    }
    //写一个方法，用于工作
    public static void doWork(){
        for (IUSB usb : usbs) {
            usb.swapData();
        }
    }
}

    多态

    2017-03-25
    Java

    概念：

    多种形态，是指父类或接口定义的引用变量，可以只想子类或者实现子类的实例对象，在调用方法时，是调用该引用变量所指向的实例对象。

    满足多态的三个条件：

    1. 继承
    2. 重写
    3. 父类引用指向子类实例对象

    注意：父类引用对象不能使用子类新增方法

    父类与子类关系

    • 父类是父类
    • 子类是父类
    • 子类是子类
    • 子类不是其他类
    • 父类不是子类
    EX:例子
    关系

    父类是父类Animal animal = new Animal();

    子类是子类Dog dog = new Dog(); Cat cat = new Cat();

    子类是父类(父类引用指向子类对象)
    Animal animal11 = new Dog();
    animal11.eat();
    Animal animal12 = new Cat();
    animal12.eat();

    父类不是子类Dog dog1 = new Animal();

    子类不是其他类Cat cat1 = new Dog();

    例问题：

    饲养员：提供一个喂食的方法不存在多态的前提：饲养员类中需要存在多个喂食方法，面向不同的动物，提供不同喂食使用，饲养员中只需要使用一个喂食方法

    遗留问题：

    使用多态后，无法调用子类的新增对象方法

    解决方案：instanceof: 判断对象是否是某种类型，boolean ret = 对象 instanceof 类型，左边对象是右边类型，则返回true，否则是false

    类对象

    2017-03-22
    Java

    类成员（static 修饰成员）的特点：

    1. 随着类被加载进JVM 而同时也初始化和内存中分配空间
    2. 优先于对象存在（对象也是通过new 出来的）
    3. 被该类的所有对象所共享
    4. 直接使用类名调用即可

    访问规则：

    1. 静态方法：只能访问静态的成员变量/方法 不能访问非静态的成员变量/方法
    2. 非静态方法可以访问非静态成员变量/方法 也可以访问静态的成员变量/方法
    3. 对象可以访问静态成员、但是不推荐、因为底层依然是同类名在调用
    4. 类可以访问静态成员、类不能访问非静态成员（成员变量/方法）

    类成员和实例成员的访问：类、成员：使用static 修饰的成员变量/方法

    一般在开发中类成员如何使用：

    • 一般在开发中工具类的工具方法都使用static 修饰,如此的好处–> 直接使用（工具类名，方法名）
    • 比如说，工具类：java.util.Array中的方法都是用Static修饰

    变量的分类和初始值：

    变量因定义的位置不同，分为: 成员变量和局部变量

    • 成员变量：直接定义在类中的花括号中的变量，有称为字段。也有人称为属性（尽量说字段），有些人说成员变量不对。
    • 局部变量：除了成员变量，就是局部变量
      > 1. 直接定义在方法中的变量
> 2. 方法形式参数
> 3. 代码块中（{}）变量

    区别：

    1. 成员变量都是有初始化值
    2. 局部变量没有初始化的值
    3. 使用局部变量，必须先赋值，并且变量都有默认值

    特殊：在Java中，只有final修饰符可以修饰局部变量，其他不能修饰局部变量

    类型 初始值
    byte、short、int 0
    long 0
    char
    double 0.0
    float 0.0f
    boolean false
    引用类型 null

    变量的作用域和生命周期

    当一个变量被定义的时候，他的作用域就被确定了

    • 成员变量：在整个类中有效
    • 局部变量：只在当前定义的{}中有效

    变量的类型 变量存在的地方 生命周期开始 生命周期结束
    类变量 类里 static修饰 字节码加载 字节码被卸载
    实例变量 类里无static修饰 new对象开始 对象
    局部变量 方法里，形参 方法调用开始 方法调用完结束

    局部变量的初始化和JVM的运行机制

    局部变量定义之后，必须显示初始化后才能使用，因为系统不会因为局部变量执行初始化操作，也就是定义局部变量后，系统并未为这个变量分配内存空间

    直到程序为这个变量赋值时，系统才会为局部变量分配内存，并将初始值保存到该内存中。

    注意：局部变量不属于任何类或视力，因此它总是保存在其所在方法的栈内存中

    1. 基本数据类型局部变量：直接把这个变量的值保存到该变量所对应的内存中
    2. 引用数据类型局部变量：这个变量的内存中存的是对中唯一的一块内存空间地址值。通过该地址引用到该变量实际引用堆里的对象。

    特别注意：栈内存中的变量无需系统垃圾回收。其往往随方法或代码块的运行结束而结束，栈内存运行速度远快与堆内存
