    接口

    2017-03-26
    Java

    概念

    表示一种规范，用来定义一组方法，表示某一类事务具备的功能，其实现类必须实现该功能。

    注意

    • 使用interface关键字声明
    • 接口能实例化
    • 一个类可实现（implement）多个接口（中间用逗号隔开），但只能继承一个类
    • 接口中的方法默认使用public abstract修饰
    • 接口中的成员变量public static final修饰
    • 接口中不能实现接口，但允许继承多个接口
    • 接口多继承时，实现最终接口的类必须把所有的父接口的方法实现

    多态

    2017-03-25
    Java

    概念：

    多种形态，是指父类或接口定义的引用变量，可以只想子类或者实现子类的实例对象，在调用方法时，是调用该引用变量所指向的实例对象。

    满足多态的三个条件：

    1. 继承
    2. 重写
    3. 父类引用指向子类实例对象

    注意：父类引用对象不能使用子类新增方法

    父类与子类关系

    • 父类是父类
    • 子类是父类
    • 子类是子类
    • 子类不是其他类
    • 父类不是子类

    类对象

    2017-03-22
    Java

    类成员（static 修饰成员）的特点：

    1. 随着类被加载进JVM 而同时也初始化和内存中分配空间
    2. 优先于对象存在（对象也是通过new 出来的）
    3. 被该类的所有对象所共享
    4. 直接使用类名调用即可

    访问规则：

    1. 静态方法：只能访问静态的成员变量/方法 不能访问非静态的成员变量/方法
    2. 非静态方法可以访问非静态成员变量/方法 也可以访问静态的成员变量/方法
    3. 对象可以访问静态成员、但是不推荐、因为底层依然是同类名在调用
    4. 类可以访问静态成员、类不能访问非静态成员（成员变量/方法）

    枚举

    2017-03-22
    Java

    枚举：能列举出所有情况、使用枚举，枚举类使用enum 修饰

    枚举注意事项：

    • 1、所有的枚举类继承自java.lang.Enum
    • 2、枚举类中的所有类成员均被public static final 每一个成员其实都是一个被枚举实例出来的对象
    public class EnumDemo {
    public static void main(String[] args) {
        Weekday i = Weekday.SUN;
        //name():获取名字
        System.out.println(i);
        //ordinal():代表位置，从0 开始
        System.out.println(i.ordinal());
        //values():获取枚举类中所有对象
        Weekday[] weekday = i.values();
    }
}
//枚举类型
enum Weekday {SUN,MON,TUS,WED,THU,FRI,SAT}
