    类对象

    2017-03-22
    Java

    类成员（static 修饰成员）的特点：

    1. 随着类被加载进JVM 而同时也初始化和内存中分配空间
    2. 优先于对象存在（对象也是通过new 出来的）
    3. 被该类的所有对象所共享
    4. 直接使用类名调用即可

    访问规则：

    1. 静态方法：只能访问静态的成员变量/方法 不能访问非静态的成员变量/方法
    2. 非静态方法可以访问非静态成员变量/方法 也可以访问静态的成员变量/方法
    3. 对象可以访问静态成员、但是不推荐、因为底层依然是同类名在调用
    4. 类可以访问静态成员、类不能访问非静态成员（成员变量/方法）

    类成员和实例成员的访问：类、成员：使用static 修饰的成员变量/方法

    一般在开发中类成员如何使用：

    • 一般在开发中工具类的工具方法都使用static 修饰,如此的好处–> 直接使用（工具类名，方法名）
    • 比如说，工具类：java.util.Array中的方法都是用Static修饰

    变量的分类和初始值：

    变量因定义的位置不同，分为: 成员变量和局部变量

    • 成员变量：直接定义在类中的花括号中的变量，有称为字段。也有人称为属性（尽量说字段），有些人说成员变量不对。
    • 局部变量：除了成员变量，就是局部变量
      > 1. 直接定义在方法中的变量
> 2. 方法形式参数
> 3. 代码块中（{}）变量

    区别：

    1. 成员变量都是有初始化值
    2. 局部变量没有初始化的值
    3. 使用局部变量，必须先赋值，并且变量都有默认值

    特殊：在Java中，只有final修饰符可以修饰局部变量，其他不能修饰局部变量

    类型 初始值
    byte、short、int 0
    long 0
    char
    double 0.0
    float 0.0f
    boolean false
    引用类型 null

    变量的作用域和生命周期

    当一个变量被定义的时候，他的作用域就被确定了

    • 成员变量：在整个类中有效
    • 局部变量：只在当前定义的{}中有效

    变量的类型 变量存在的地方 生命周期开始 生命周期结束
    类变量 类里 static修饰 字节码加载 字节码被卸载
    实例变量 类里无static修饰 new对象开始 对象
    局部变量 方法里，形参 方法调用开始 方法调用完结束

    局部变量的初始化和JVM的运行机制

    局部变量定义之后，必须显示初始化后才能使用，因为系统不会因为局部变量执行初始化操作，也就是定义局部变量后，系统并未为这个变量分配内存空间

    直到程序为这个变量赋值时，系统才会为局部变量分配内存，并将初始值保存到该内存中。

    注意：局部变量不属于任何类或视力，因此它总是保存在其所在方法的栈内存中

    1. 基本数据类型局部变量：直接把这个变量的值保存到该变量所对应的内存中
    2. 引用数据类型局部变量：这个变量的内存中存的是对中唯一的一块内存空间地址值。通过该地址引用到该变量实际引用堆里的对象。

    特别注意：栈内存中的变量无需系统垃圾回收。其往往随方法或代码块的运行结束而结束，栈内存运行速度远快与堆内存

    什么是异常机制

    什么是异常，比如说：
    医院–》 医生–》 身体某些部位异常–》 该部位和正常相比不同
    将受到影响

    异常：

    并不是语法出错，语法错误，无法编译，不会产生字节码文件，不能运行

    为什么学习异常机制？

    • 异常处理是衡量一门语言是否成熟的标准之一
    • java、c++，c# 等都有异常处理
    • 异常处理都可以让程序有更好的容错性，是代码更加健壮
    • if-else 其实是一种异常

      没有异常存在缺点：

    • 使用方法的返回值来表示异常情况有限，无法琼剧所有的情况
    • 异常流程和正常流程混合在一起，增大程序的复杂性，可读性也不好
    • 随着系统规模的不断扩大，程序的可维护性极低

      针对上述的没有异常机制的缺点，我们提出

    1. 把异不同类型的异常情况描述成不同类
    2. 分离异常流程代码和正常代码不一样

      异常转译：

      当位于上层的子系统不需要关心底层的异常细节时，常见的做法就是捕获异常，把它转换为新的不同类型的异常，再抛出新的异常

      异常链：

      把原始的异常包装为新的异常类，从而形成多个异常的有序排列，有助于查找异常的根本原因

    1. 使用匿名内部类时，我们必须继承一个类或者实现一个接口，但是两者不可兼得，匿名内部类必须继承或者实现一个接口

    5. 匿名内部类不能是抽象的，他必须要实现继承或者实现接口的所有抽象方法

    什么是字符串，一串字符

    本质：char[] 数组

    字符串的分类

    1、不可变的字符串：String 类
    2、可变的字符串：StringBuilder 、 StringBuffer 类

    不可变的字符串：

    一个String 对象只能表示一个固定的字符串，
    若内容改变，则对象也改变

    字符串常见问题：

    • 1、单独使用”” 引号创建的字符串都是直接量，编译器就已经确定储存到常量池中
    • 2、使用new String() 创建的对象会存到堆内存中，是运行期才会有
    • 3、使用只包含直接量的字符串链接符，如”aa”+”bb” 创建的也是直接量编译期就能确定，已经确定存储到常量池中
    • 4、使用String 直接量（无final 修饰）的字符串表达式如”aa”+temp 创建的对象试运行期才创建得存储在堆中的0

    String中的常用方法:
