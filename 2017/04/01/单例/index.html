    单例模式

    2017-04-01
    Java

    概念

    单例：即只有一个实例对象

    单例的模式：

    • 饿汉式：效率低、线程安全、在第一次加载类时自动创建实例对象
    • 懒汉式：第一次调用实例对象时，将实例对象创建并返回，线程不安全，效率高

    如何实现单例：

    1. 构造器私有化
    2. 提供一个私有的静态的成员变量，并且自行创建该实例对象
    3. 提供一个静态的公共的访问方法、将该实例对象返回调用者

    为了保护线程安全，使用synchronized

    饿汉式

    提供一个静态的、私有的、最终的该实例对象，并且赋值，这种方法在某些特定的情况下，占用内存

    public class SingleDemo01 {
    public static void main(String[] args) {

    }
}
class Demo01{
    //提供一个私有的静态的成员变量，并且自行创建该实例对象
    private static Demo01 instance = new Demo01();
    //构造器私有化
    private Demo01(){
    }
    //提供一个公共的静态的访问方法，将该实例对象返回给调用者
    public static Demo01 getInstance(){
        return instance;
    }
}

    懒汉式

    提供一个私有的，静态的实例对象，当时不赋值，等待调用时赋值。

    public class SingleDemo02 {
    public static void main(String[] args) {
        Demo02 demo01 = Demo02.getInstance();
        Demo02 demo02 = Demo02.getInstance();
        System.out.println(demo01==demo02);
    }
}
class Demo02{
    //构造器私有化
    private Demo02(){

    }
    /*
    * 提供一个私有的静态的实例化对象的引用，但此时不创键对象、仅仅声明
    * */
    private static Demo02 instance = null;
    //提供一个静态的私有的
    public static Demo02 getInstance(){
        if (instance==null){
            instance = new Demo02();
        }
        return instance;
    }
}

    懒汉式改进

    懒汉模式有线程安全问题，将初始化该例对象放入synchronized同步代码块

    class SingletonTest4 {
    //定义一个私有的构造方法
    public SingletonTest4() {

    }

    //将自身的实例对象设置为一个属性
    //并加上static
    //注意不使用final
    //加上volatile 保证多线程访问时候
    /*
    * 第一、加上volotitle 保证多线程访问时候instance 变量的可见性
    * 第二、避免instance 初始化的时候其他变量还未赋值完成的时候，被另外的线程调用
    *
    * */
    private static volatile SingletonTest4 instance;

    //静态方法返回该类的实例
    //什么时候调用什么时候实例化
    //问题：在多线程访问时，会造成重复初始化
    public static SingletonTest4 getInstance() {
        //对象实例化时判断与否
        if (instance==null){
            //同步代码块（对象未初始化的时候，使用同步代码块），保证我们多线程访问的时候对象在第一次创建后，不再重复创建
            synchronized (SingletonTest4.class){
                //未初始化，则初始instance 变量
                if (instance==null){
                    instance = new SingletonTest4();
                }
            }
        }
        return instance;
    }
}
