    策略设计模式

    2017-03-30
    Java

    策略设计模式

    软件的设计模式，指的是由某个行为，但是在某个场景中，该行为有不同的实现算法。

    实现方法

    • 定义一个接口，创建功能方法。
    • 各个策略类实现接口，并在功能方法中实现各自不同的逻辑。
    • 在调用者类中创建一个接口对象，重写调用者构造器，将传进来的策略类的对象赋值给接口对象，并在之后的功能方法中调用策略类对象的功能方法

    比如：每个人都得交税：

    —–>美国税 %4

    —–>中国 %20

    不一样—–> 税收算法不一样

    策略模式的实现

    1. 什么是策略

      定义了不同的算法，并且之间可以相互替换

    2. 策略模式的喊出

      可以动态的改变对象的行为

    3. 策略设计原则

      把一个类中经常改变过着将来可能改变的部分提取出来

    优势：

    采用策略模式可以避免由于使用条件语句带来的代码混乱，提高应用程序的灵活性、条理性
    可以随意的调用实现了接口的类的行为

    开发中应用场景：

    当一个用用程序需要是想一种特定的服务或者功能，而且该程序有多种实现方法的时候使用
