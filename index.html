    单例模式

    2017-04-01
    Java

    概念

    单例：即只有一个实例对象

    单例的模式：

    • 饿汉式：效率低、线程安全、在第一次加载类时自动创建实例对象
    • 懒汉式：第一次调用实例对象时，将实例对象创建并返回，线程不安全，效率高

    如何实现单例：

    1. 构造器私有化
    2. 提供一个私有的静态的成员变量，并且自行创建该实例对象
    3. 提供一个静态的公共的访问方法、将该实例对象返回调用者

    为了保护线程安全，使用synchronized

    饿汉式

    提供一个静态的、私有的、最终的该实例对象，并且赋值，这种方法在某些特定的情况下，占用内存

    public class SingleDemo01 {
    public static void main(String[] args) {

    }
}
class Demo01{
    //提供一个私有的静态的成员变量，并且自行创建该实例对象
    private static Demo01 instance = new Demo01();
    //构造器私有化
    private Demo01(){
    }
    //提供一个公共的静态的访问方法，将该实例对象返回给调用者
    public static Demo01 getInstance(){
        return instance;
    }
}

    懒汉式

    提供一个私有的，静态的实例对象，当时不赋值，等待调用时赋值。

    public class SingleDemo02 {
    public static void main(String[] args) {
        Demo02 demo01 = Demo02.getInstance();
        Demo02 demo02 = Demo02.getInstance();
        System.out.println(demo01==demo02);
    }
}
class Demo02{
    //构造器私有化
    private Demo02(){

    }
    /*
    * 提供一个私有的静态的实例化对象的引用，但此时不创键对象、仅仅声明
    * */
    private static Demo02 instance = null;
    //提供一个静态的私有的
    public static Demo02 getInstance(){
        if (instance==null){
            instance = new Demo02();
        }
        return instance;
    }
}

    懒汉式改进

    懒汉模式有线程安全问题，将初始化该例对象放入synchronized同步代码块

    class SingletonTest4 {
    //定义一个私有的构造方法
    public SingletonTest4() {

    }

    //将自身的实例对象设置为一个属性
    //并加上static
    //注意不使用final
    //加上volatile 保证多线程访问时候
    /*
    * 第一、加上volotitle 保证多线程访问时候instance 变量的可见性
    * 第二、避免instance 初始化的时候其他变量还未赋值完成的时候，被另外的线程调用
    *
    * */
    private static volatile SingletonTest4 instance;

    //静态方法返回该类的实例
    //什么时候调用什么时候实例化
    //问题：在多线程访问时，会造成重复初始化
    public static SingletonTest4 getInstance() {
        //对象实例化时判断与否
        if (instance==null){
            //同步代码块（对象未初始化的时候，使用同步代码块），保证我们多线程访问的时候对象在第一次创建后，不再重复创建
            synchronized (SingletonTest4.class){
                //未初始化，则初始instance 变量
                if (instance==null){
                    instance = new SingletonTest4();
                }
            }
        }
        return instance;
    }
}

    代理模式

    2017-03-30
    Java

    概念

    代理模式，首先定义一个接口，代理者与委托者都实现这个接口，代理类中首先实例化一个接口类型的对象但不赋值（用于实现委托者功能），在代理者构造器中将其赋值，代理者的功能方法中，实现的是委托者方法.

    //抽象的主题，定义主要功能
interface Subject {
    public void operate();
}

class RealStudent implements Subject {
    @Override
    public void operate() {
        System.out.println("学习生物科学");
    }
}

class RealStudentTwo implements Subject {
    @Override
    public void operate() {
        System.out.println("学习化学");
    }
}

class RealStudentThree implements Subject {
    String name;

    @Override
    public void operate() {
        System.out.println("学java");
    }
}

class Praxy implements Subject {
    private Subject student;

    public Praxy(Subject subject) {
        this.student = subject;
    }

    @Override
    public void operate() {
        System.out.println("before operate");
        student.operate();
        System.out.println("end operate");
    }
}

public class DelegateDemo {
    public static void main(String[] args) {
        Subject subject = new RealStudent();
        Subject subject1 = new RealStudentTwo();
        Subject subject2 = new RealStudentThree();
        Praxy praxy = new Praxy(subject2);
        praxy.operate();
    }

    策略设计模式

    2017-03-30
    Java

    策略设计模式

    软件的设计模式，指的是由某个行为，但是在某个场景中，该行为有不同的实现算法。

    实现方法

    • 定义一个接口，创建功能方法。
    • 各个策略类实现接口，并在功能方法中实现各自不同的逻辑。
    • 在调用者类中创建一个接口对象，重写调用者构造器，将传进来的策略类的对象赋值给接口对象，并在之后的功能方法中调用策略类对象的功能方法

    比如：每个人都得交税：

    —–>美国税 %4

    —–>中国 %20

    不一样—–> 税收算法不一样

    策略模式的实现

    1. 什么是策略

      定义了不同的算法，并且之间可以相互替换

    2. 策略模式的喊出

      可以动态的改变对象的行为

    3. 策略设计原则

      把一个类中经常改变过着将来可能改变的部分提取出来

    优势：

    采用策略模式可以避免由于使用条件语句带来的代码混乱，提高应用程序的灵活性、条理性
    可以随意的调用实现了接口的类的行为

    开发中应用场景：

    当一个用用程序需要是想一种特定的服务或者功能，而且该程序有多种实现方法的时候使用

    继承的关系

    在原有类的继承之上，加以拓展，从而产生新的子类的过程。子类其实就是父类的一种特殊情况。

    • 狗继承动物，狗是一种特殊的动物
    • 猫继承动物，猫是一种特殊的动物


        class Animal{
            void sleep(){
                System.out.println(“zZZ…”);
            }
        }
        class Cat{
            void catchMouse(){
                System.out.println(“捉老鼠”);
            }
        }
        class Dog{
            void watch(){
                System.out.println(“看门”);
            }
        }
        public static void main(String[] args){
            Animal a = new Animal();
        }

    继承的语法格式

    class 子类名 extends 父类名{}

    子类可以继承到父类哪些成员：

    1. 如果使用的public 或者protected 修饰。子类可以继承
    2. 如果使用的是包访问权限（不写），如果父类与子类在同一个包下，可以继承，如果不在同一包下，不能继承
    3. 父类的构造器，子类无法继承
    4. 父类用private修饰，则子类不能继承
    ex:
    Teacher teacher = new person();        //不可以！！！
Person person = new Teacher();        //可以

    注意：
    在java中的继承，只允许单继承，不允许继承多个类，但允许多重继承。

    This关键字

    this 当前对象，哪一个对象调用了this所在的成员，this就是哪一个对象

    this的使用场景：
    1. 解决成员变量和局部变量的二义性，必须使用setter方法
    2. 同一个类中多个实例方法间互调
    3. 将当前对象作为参数传递给另一个方法
    4. 将当前对象作为方法的返回值
    5. 构造器重载的相互作用，this必须写在第一行（this（）表示的就是无参数的构造器）
    6. static不能和this一起使用

    方法覆盖

    1. 子类拓展父类的时候，某个方法不适合子类本身
    2. 方法重载Overload：两同一不同
    3. 方法覆盖Override：一同两小一大
      • 一同：方法的名称相同
      • 两小
        • 子类方法的返回类型小于父类方法的返回类型
        • 子类方法不能抛出新的异常
      • 一大：子类方法的修饰符必须大于父类方法的修饰符

    王道的做法：

    直接复制父类的方法定义到子类。重新定义方法体

    判断是否覆盖的准则：

    java5后，用@Override判断方法是否覆盖

    注意：

    方法的覆盖、重载、只能对方法有效，类、字段、构造器没有覆盖的概念

    super关键字

    • this关键字：当前对象
    • super关键字：当前对象的父类对象
    super的应用场景：
    1. 在子类方法中，调用父类被覆盖的方法
    2. 在子类方法中，返回父类被隐藏的成员变量（极少使用）
    3. 在子类构造器中，调用父类的构造器父类的构造器，super([参数])
    子类的创建过程：
    1. 创建子类对象时，会调用子类的构造器，而再调用子类构造器之前，会先调用父类的构造器
    2. 先对父类做初始化，再对子类做初始化

    隐藏问题

    遮蔽：把之前的挡住了

    1. 局部变量和成员变量同名，局部变量隐藏成员变量，使用this关键字解决
    2. 在满足继承条件下（很少见）子类存在和弗雷同名的成员变量，子类变量隐藏了父类变量，使用super关键字解决
    3. 在满足继承条件下（很少见）子类存在和父类相同的静态方法，静态方法不存在覆盖

    Object便是对象的意思，Object类就是对象的封装体，Object可以表示所有对象
    所有对象，包括了数组都实现了Object类中的方法

    接口

    2017-03-26
    Java

    概念

    表示一种规范，用来定义一组方法，表示某一类事务具备的功能，其实现类必须实现该功能。

    注意

    • 使用interface关键字声明
    • 接口能实例化
    • 一个类可实现（implement）多个接口（中间用逗号隔开），但只能继承一个类
    • 接口中的方法默认使用public abstract修饰
    • 接口中的成员变量public static final修饰
    • 接口中不能实现接口，但允许继承多个接口
    • 接口多继承时，实现最终接口的类必须把所有的父接口的方法实现
    public class InterfaceDemo03 {
    public static void main(String[] args) {
        //提供键盘对象、鼠标对象
        MainBoard.plugin(new Mouse());
        MainBoard.plugin(new KeyBoard());
        MainBoard.doWork();
    }
}
interface IUSB{
    //提供一个交换数据
    void swapData();
}
//鼠标
class Mouse implements IUSB{
    @Override
    public void swapData() {
        System.out.println("鼠标在移动。。。");
    }
}
//硬盘
class KeyBoard implements IUSB{

    @Override
    public void swapData() {
        System.out.println("敲打键盘。。。");
    }
}
//主板
class MainBoard{
    //声明成员变量
    private static IUSB[] usbs = new IUSB[2];
    //声明一个控制当前下标的变量
    private static int index = 0;
    //写一个静态方法，用于将鼠标、键盘插入
    public static void plugin(IUSB iusb){
        if (usbs.length == index){
            System.out.println("USB满了");
            return;
        }
        usbs[index] = iusb;
        index++;
    }
    //写一个方法，用于工作
    public static void doWork(){
        for (IUSB usb : usbs) {
            usb.swapData();
        }
    }
}

    多态

    2017-03-25
    Java

    概念：

    多种形态，是指父类或接口定义的引用变量，可以只想子类或者实现子类的实例对象，在调用方法时，是调用该引用变量所指向的实例对象。

    满足多态的三个条件：

    1. 继承
    2. 重写
    3. 父类引用指向子类实例对象

    注意：父类引用对象不能使用子类新增方法

    父类与子类关系

    • 父类是父类
    • 子类是父类
    • 子类是子类
    • 子类不是其他类
    • 父类不是子类
    EX:例子
    关系

    父类是父类Animal animal = new Animal();

    子类是子类Dog dog = new Dog(); Cat cat = new Cat();

    子类是父类(父类引用指向子类对象)
    Animal animal11 = new Dog();
    animal11.eat();
    Animal animal12 = new Cat();
    animal12.eat();

    父类不是子类Dog dog1 = new Animal();

    子类不是其他类Cat cat1 = new Dog();

    例问题：

    饲养员：提供一个喂食的方法不存在多态的前提：饲养员类中需要存在多个喂食方法，面向不同的动物，提供不同喂食使用，饲养员中只需要使用一个喂食方法

    遗留问题：

    使用多态后，无法调用子类的新增对象方法

    解决方案：instanceof: 判断对象是否是某种类型，boolean ret = 对象 instanceof 类型，左边对象是右边类型，则返回true，否则是false
