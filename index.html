    1. xml是什么

    HTML

    1.什么是html

    是超文本标记语言，它规定了自己的语法规则，用来表示比“文本”更丰富的意义，比如图片，表格，链接等。浏览器软件知道HTML语言的语法，目前互联网上的绝大部分网页都是使用HTML编写的。

    2.html的结构

    文件名：xxx.html或者xxx.htm，但是通常用xxx.html。

    <!DOCTYPE HTML>：html5的文档声明 
<html>：所有的html都要有这个标签，表示文档的开始和结束
<head>：用来存放html中的基本属性信息，比如标题，编码等
<body>：用来存放html中的页面数据，在页面中是可以显示的

<title>：指定网页的标题
<meta charset=UTF-8/>：指定浏览器的解码格式

    单例模式

    2017-04-01
    Java

    概念

    单例：即只有一个实例对象

    单例的模式：

    • 饿汉式：效率低、线程安全、在第一次加载类时自动创建实例对象
    • 懒汉式：第一次调用实例对象时，将实例对象创建并返回，线程不安全，效率高

    如何实现单例：

    1. 构造器私有化
    2. 提供一个私有的静态的成员变量，并且自行创建该实例对象
    3. 提供一个静态的公共的访问方法、将该实例对象返回调用者

    为了保护线程安全，使用synchronized

    饿汉式

    提供一个静态的、私有的、最终的该实例对象，并且赋值，这种方法在某些特定的情况下，占用内存

    策略设计模式

    2017-03-30
    Java

    策略设计模式

    软件的设计模式，指的是由某个行为，但是在某个场景中，该行为有不同的实现算法。

    实现方法

    • 定义一个接口，创建功能方法。
    • 各个策略类实现接口，并在功能方法中实现各自不同的逻辑。
    • 在调用者类中创建一个接口对象，重写调用者构造器，将传进来的策略类的对象赋值给接口对象，并在之后的功能方法中调用策略类对象的功能方法

    比如：每个人都得交税：

    —–>美国税 %4

    —–>中国 %20

    不一样—–> 税收算法不一样

    模板方法

    2017-03-30
    Java

    模板方法设计模式：

    在父类的一个方法中定义一个总体的算法骨架，而将某一些步骤延迟到子类中去，不同的子类实现的细节不同，模板方法是的子类可以不改变父类的算法结果，而使得算法的细节步骤改变，抽象父类负责定义操作中的总体的算法骨架，某些具体实现步骤由子类决定

    注意：
    抽象的父类，只是定义了一个通用的算法，其实现必须依赖子类的辅助，模板方法作为模板样式不准子类复写
